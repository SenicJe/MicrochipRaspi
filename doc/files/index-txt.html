Introduction

The 3DTouchPad SDK is a software development kit bringing interaction with Microchip 3DTouchPad devices.  It provides a high level API for the communication and the use of the features of the device.  It also contains this manual and example applications which demonstrate how to use the API in different contexts.

This version of the SDK contains binaries for Windows and the source code that builds for Linux without modification.

The page Compilation has instructions on how to compile the SDK on the different platforms.  API Usage, First Steps, Data Retrieval, Real Time Control, 3D Firmware Update and 2D Firmware Update contain instructions on how to use the pre-built binaries inside an application.

The primary task of the MGC3130 is detection of Position and/or Gestures.
Real Time Control (RTC) functions allow to change HMI parameters during runtime and trigger special actions.
Update of 2D firmware requires a connection to the 2D bootloader device.
The 2draw application shows a ANSI-art heat-map of 2D mutual, calibrated sensor data.
The Console Application retrieves more data than the Stream Example and allows to trigger a few Real Time Controls.
Displays stream of 2D finger positions.
The keycombo application demonstrates how to change the codes of the key-presses that will be emulated by the 3DTouchPad on gesture events.
The programmer application demonstrates how to update the Loader Library and the GestIC Library.
The dynamic stream example demonstrates how to use the Dynamic API to use the API when it is provided as an DLL.
The static stream example demonstrates how the Static API could be used if the application is built together with the API or if it is linked statically against the API.
Demonstrates the 2D Firmware Update based on HEX-files.
The MGC Image Conversion tool (MGCConv) allows extraction of firmware-images as C-structures from ENZ-Files.
The CDC-Driver is required for communication with Hillstar or Sabrewing-devices via USB-Bridge.
