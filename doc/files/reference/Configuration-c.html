Macros

HMI_CUSTOM

Triggers the inclusion of hmi_custom.h into the API.

This macro has to be provided via the command line arguments of the compiler.  If defined hmi_api.h includes hmi_custom.h which has to be at one of the include paths.

hmi_custom.h can override the default configuration and provide an application specific custom API.

Important

Both the API and the application have to be compiled with the same hmi_custom.h in place.  Otherwise it will show undefined behavior.

HMI_IO

HMI_IO allows to set or override the IO-implementation.

HMI_IO_CUSTOMThe application provides its own implementation of the IO
HMI_IO_CDC_SERIALConnects to the MGC3130 via the IÂ²C to USB Bridge.

When HMI_IO is set to HMI_IO_CUSTOM make sure to define the correct synchronization mode for message handling if hmi3d_message_handle is called outside of hmi_message_receive (the synchronization modes are HMI_SYNC_INTERRUPT and HMI_SYNC_THREADING).

HMI_API_DYNAMIC

Has to be set when compiling the API as a DLL or when using it as DLL.  This macro has to be provided to the compiler as an command line argument.

HMI_HAS_DYNAMIC

If this macro was defined in the compilers command line or inside hmi_custom.h the Dynamic API will be available even when using the Static API.

HMI_API_EXPORT

This macro has to be only used when compiling the API as a DLL.  When using the API as a DLL or linking statically against this API this macro should be absent.  It has to be provided to the compiler as an command line argument.

Macros

HMI_SYNC_INTERRUPT

This macro specifies that hmi3d_message_handle is called during an interruption of the normal operation.  This is typically done from within a low priority ISR to avoid message-loss while performing time consuming operations.

See also

HMI_SYNC_THREADING

HMI_SYNC_THREADING

This macro specifies that hmi3d_message_handle is called from within a different thread than the normal API functions.  Here a bidirectional synchronization is needed to avoid race conditions.

Threading IO mode allows to do IO completely asynchronously from the rest of the application.
