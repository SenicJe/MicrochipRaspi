Data Retrieval

The primary task of the MGC3130 is detection of Position and/or Gestures.  The result of those detections could be fetched with the data retrieval feature of the SDK.

The 3DTouchPad consists of the 2D and 3D subsystems which are on separated chips.  To allow maximum performance the API provides separated functions to communicate with each subsystem.

3D Data Output

New data could be retrieved with a call to hmi3d_retrieve_data which updates the internal result buffers.

The buffers for the individual data could be get with the hmi3d_get_* functions.  The content of those buffers are updated when calling hmi3d_retrieve_data when there were new values or events.  What kind of data gets communicated could be controlled with hmi3d_set_output_enable_mask.

2D Data Output

2D data output works analog to the 3D data output.

A call to the function hmi2d_retrieve_data updates the internal buffer that could be accessed via the pointers returned by the corresponding hmi2d_get_ functions.

The possible data buffers and their access functions are:

FunctionType[Data
hmi2d_get_mutual_rawhmi2d_block_tRaw mutual sensor data
hmi2d_get_mutual_calhmi2d_block_tCalibrated mutual sensor data
hmi2d_get_self_rawhmi2d_row_tRaw self sensor data
hmi2d_get_self_calhmi2d_row_tCalibrated self sensor data
hmi2d_get_finger_positionshmi2d_finger_pos_list_tList of finger positions
hmi2d_get_mousehmi2d_mouse_tState of mouse emulation
hmi2d_get_gesturehmi2d_gesture_tGestures as used for keyboard emulation
