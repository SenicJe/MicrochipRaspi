Introduction

After Power-on or hardware Reset the MGC3130 executes the Library Loader routine.  First it sends a Fw_Version_Info message to the host and starts a timer.  After a time-out of 100 ms, the GestIC Library is started automatically.

Before the timer timed-out the GestIC Library update can be initiated via the Fw_Update_Start message.  Once started the Library Loader stays in update mode until the next Reset of the MGC3130.

The update itself is started with the Fw_Update_Start message followed by a sequence of FW_Update_Block message and finished with a Fw_Update_Complete message.

An update of the Library Loader itself gets done by replacing the GestIC Library with a Library Loader Updater.  After the next reset this Updater is started instead of the GestIC Library and updates the Library Loader.  During this procedure the Library Loader Updater sends multiple Fw_Version_Info messages with feedback about the progress.

Afterwards the Updater has to be replaced with the GestIC Library again to have a fully operational system.  The following section describes how this process could be done with the help of the HMI API.

Acquiring Firmware Version

The currently running firmware could get fetched with hmi3d_query_fw_version.

char fw_version[120];
int error;

error = hmi3d_query_fw_version(hmi, fw_version, sizeof(fw_version), 100);
if(error != HMI_NO_ERROR) {
    fprintf(stderr, "Error: Could not read firmware version\n");
    return error;
}

printf("Currently running GestIC Library: %s\n", fw_version);

Using hmi3d_update_image

The easiest way to do an update is calling hmi3d_update_image.  This function does the whole procedure of updating the firmware from a first Reset to sending the final Fw_Update_Complete message with the instruction to do one more Reset.

hmi3d_update_image expects an instance of hmi3d_update_image_t as an argument which contains the image to be flashed.  This image can be extracted from ENZ-files with the help of MGCConv.

MGCConv is a tool which can create self-contained source-files with the definition of a global variable containing the image.  The structure used in this declarations is compatible with hmi3d_update_image_t.  Therefore this variables could easily be used by forward declaring those variables as extern or by casting them to hmi3d_update_image_t.  This allows to simply add those source-files to a project and use the contained images to update the firmware.

// Update of the GestIC Library alone.

extern hmi3d_update_image_t Library;

int flash_image(hmi_t *hmi)
{
    int session_id = 1; // Some random value
    int error;
    error = hmi3d_update_image(&hmi, session_id, &Library,
                               hmi3d_UpdateFunction_ProgramFlash, 100);
    if(error != HMI_NO_ERROR)
    {
        fprintf(stderr, "Error: Could not flash image\n");
        return error;
    }
    return HMI_NO_ERROR;
}

Updating the Library Loader requires one call to hmi3d_update_image to write the Library Loader Updater.  Before continuing with the update of the GestIC Library the application has to wait until the Updater has updated the Library Loader which requires about 20 seconds.

The function hmi3d_update_wait_loader_done blocks until the Library Loader Updater has finished the update and sent the appropriate Fw_Version_Info message.

Afterwards a second calls to hmi3d_update_image is needed to replace the Updater with the GestIC Library again.

// Complete update including the Library Loader and the GestIC Library

extern hmi3d_update_image_t Loader;
extern hmi3d_update_image_t Library;

int flash_image(hmi_t *hmi)
{
    int session_id = 1; // Some random value
    int error;
    error = hmi3d_update_image(&hmi, session_id, &Loader,
                               hmi3d_UpdateFunction_ProgramFlash, 100);
    if(error != HMI_NO_ERROR)
    {
        fprintf(stderr, "Error: Could not flash Library Loader Updater\n");
        return error;
    }

    error = hmi3d_update_wait_loader_done(&hmi, 20000);
    if(error != HMI_NO_ERROR)
    {
        fprintf(stderr, "Error: Library Loader Updater failed to update Loader\n");
        return error;
    }

    session_id = 2;
    error = hmi3d_update_image(&hmi, session_id, &Library,
                               hmi3d_UpdateFunction_ProgramFlash, 100);
    if(error != HMI_NO_ERROR)
    {
        fprintf(stderr, "Error: Could not flash GestIC Library\n");
        return error;
    }
    return HMI_NO_ERROR;
}

Sending messages directly

The disadvantage of hmi3d_update_image is that it blocks until the flash process was finished without any progress indicator.  An alternative is to use the underlying functions directly.

hmi3d_update_begin starts the flash process by resetting the device and sending the Fw_Update_Start message.  The individual blocks could be sent by a call to hmi3d_update_write.  hmi3d_update_end finalizes the flash process by sending Fw_Update_Completed twice: Once with the instruction to store the new Firmware version and once with the instruction to do a reset.

Note

The reset triggered by hmi3d_update_image or hmi3d_update_end is executed after a delay of several seconds.

The implementation of hmi3d_update_image or the source-code of the Programmer Application could be used as a reference of how to use the low-level hmi3d_update_* functions directly.

int hmi3d_update_image(hmi_t *hmi,
                       unsigned int session_id,
                       hmi3d_update_image_t *image,
                       hmi3d_UpdateFunction_t mode)
{
    int error = HMI_NO_ERROR;
    int i;
    hmi3d_update_record_t *record;

    error = hmi3d_update_begin(hmi, session_id, image->iv, mode);

    for(i = 0; !error && i < image->record_count; ++i) {
        record = image->data + i;
        error = hmi3d_update_write(hmi, record->address, record->length,
                                   record->data, mode);
    }

    if(!error)
        error = hmi3d_update_end(hmi, image->fw_version);

    return error;
}

Verification

The hmi3d_update_* functions have a second mode of operation: hmi3d_UpdateFunction_VerifyOnly.  In this mode the records are send for verification of the existing firmware only.

If a session was started by providing hmi3d_UpdateFunction_VerifyOnly to hmi3d_update_begin no call to hmi3d_update_write with a different update function is allowed.

However it is possible to call hmi3d_update_write with hmi3d_UpdateFunction_VerifyOnly during a session started with hmi3d_UpdateFunction_ProgramFlash.  This could be used to verify the data on the chip immediately after writing them.
