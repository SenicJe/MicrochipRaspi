Real Time Control

Real Time Control (RTC) functions allow to change HMI parameters during runtime and trigger special actions.

Like the Data Retrieval the RTC support is also split for the two subsystem.

3D Real Time Control

Parameters could be read with hmi3d_get_param and hmi3d_set_param.  hmi3d_parameter_id_t is an enumeration of the available parameters.  A complete description of them could be found in the MGC3130 GestIC Library Interface Description User’s Guide.

While all parameters could be controlled with hmi3d_get_param and hmi3d_set_param some parameters have special helper functions with an easier to use signature.  Those parameters are:

The parameters hmi3d_param_trigger, hmi3d_param_makePersistent and hmi3d_param_dataOutputRequestMask are special in that they don’t map to an value but setting them triggers some event.

Calling hmi3d_trigger_action or setting parameter hmi3d_param_trigger with a value from the enumeration hmi3d_trigger_id_t causes the corresponding event.

Calling hmi3d_make_persistent or setting parameter hmi3d_param_makePersistent stores all parameters of the hmi3d_param_category provided as argument to the permanent flash memory.  Without this call parameters would be set back to their initial value after the next reset.

Setting the parameter hmi3d_param_dataOutputRequestMask triggers additional output of the data provided as arguments.  Unlike hmi3d_param_dataOutputEnableMask or hmi3d_param_dataOutputLockMask it affects only the next hmi3d_msg_Sensor_Data_Output message sent after receiving the parameter.

2D Real Time Control

As only the 2D chip has a direct connection to the host it controls the communication the host and the 3D chip.  Therefore the parameter for controlling which subsystems are available is part of the 2D configuration.

Likewise the emulation of mouse and keyboard is done on the 2D subsystem.  For this reason the configuration of keyboard-event emitted on the 3D flick gestures is also part of the 2D configuration.

All of the key-combination parameters could be set with the hmi2d_set_key_combo function.  It sets all four parameters related to one key-combination at once and expects the first of the first of those parameters to identify the combination.

The enumeration hmi2d_parameter_id_t maps all parameter names to their value.
